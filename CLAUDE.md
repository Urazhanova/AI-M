# CLAUDE.md — Инструкции для агента-трекера

Это рабочий файл агента, обслуживающего трекер обучения AI Mindset.  
Читается при каждом старте сессии. Содержит правила, соглашения, процедуры.

---

## Что такое этот трекер

Система для отслеживания прохождения курса AI Mindset тремя студентами:
- **Вадим** (`students/vadim/`)
- **Ирина** (`students/irina/`)
- **Мария** (`students/maria/`)

Курс разбит на **чанки** — единицы учебного контента. Каждый чанк = папка в `course/` с двумя файлами:
- `content.md` — содержательный конспект сессии
- `meta.md` — структурированные метаданные (KSA-outcomes, Bloom, DoD, граф связей)

---

## Структура папок

```
AI-M/
├── CLAUDE.md              # этот файл
├── index.md               # авто-обновляемый индекс всех материалов
├── mindset-map.md         # глобальная карта mindset-сдвигов всех студентов
├── inbox/                 # входящие транскрипты (DROP ZONE)
├── raw/                   # обработанные транскрипты-источники
├── course/
│   ├── sequence.md        # граф последовательности чанков
│   └── [chunk-id]/
│       ├── content.md
│       └── meta.md
└── students/
    └── [name]/
        ├── profile.md     # профиль студента
        ├── my-path.md     # индивидуальный путь через граф чанков
        └── progress.md    # лог прогресса (append-only)
```

## Процедура обработки новых транскриптов (inbox → raw)

**При старте каждой сессии:**
1. Проверить папку `inbox/` на наличие новых файлов
2. Если есть файлы — обработать каждый:
   a. Прочитать транскрипт
   b. Определить, к какому чанку относится (или это новый чанк)
   c. Создать / обновить `course/[chunk-id]/content.md`
   d. Создать / обновить `course/[chunk-id]/meta.md`
   e. Переместить файл в `raw/` (или попросить пользователя переместить — файловая система может быть read-only для удаления)
3. Обновить `index.md` и `course/sequence.md`

**Правила именования чанков:**
- Формат: `s{sprint}-w{week}-{type}[-suffix]`
- Примеры: `s3-w1-lecture`, `s3-w1-practical`, `s3-w2-lecture`
- Тип: `intro`, `lecture`, `practical`, `workshop`

**Если два транскрипта — части одной сессии:**
- Склей в один файл в `raw/` с суффиксом `-merged`
- Исходники сохрани с оригинальными именами в `raw/`
- В `content.md` укажи: `Источник: raw/[merged-file].md (склейка двух файлов)`

[ссылка на следующий чанк или задачу]

## Правила обновления mindset-map.md

- Обновлять статусы студентов после каждой сессии, где есть attitude-outcome
- Статусы: `—` / `seeded` / `emerging` / `internalized`
- При добавлении нового чанка с attitude-компонентом: добавить строку в таблицу
- Личные заметки — краткие, фактические, на основе рефлексий студента

---

## Правила обновления index.md

`index.md` — авто-обновляемый индекс. Обновлять при:
- Добавлении нового чанка
- Изменении статуса DoD у студента
- Добавлении нового студента

---

## Git-дружественность

Все файлы проектируются для чистых диффов:
- `progress.md` — только append (новые записи в конец)
- `mindset-map.md` — только изменение значений в таблице (не перестройка)
- `content.md` и `meta.md` — могут редактироваться, но это осознанное изменение

---

## Мультиюзерный режим (будущий GitHub)

Структура уже готова к переходу на GitHub:
- Каждый студент работает в своей папке `students/[name]/`
- Конфликты merge только в `index.md` и `mindset-map.md` (редкие)
- Рекомендация при переходе: каждый студент — отдельная ветка, merge через PR

## Сценарий учебной сессии с чанком

Когда студент говорит "начнём работу с чанком [id]" или аналогично:

**Первым делом — определить студента.** Если имя не указано явно, спросить:
> "С кем работаем сегодня — Вадим, Ирина или Мария?"

Запомнить ответ и использовать `students/[name]/` на протяжении всей сессии.

### Шаг 1 — Персонализированная подача материала

Перед любой проверкой знаний: прочитать `content.md` чанка и `students/[name]/profile.md`.  
Переписать ключевые идеи чанка в виде адаптированного текста для этого студента:

- **Язык и стиль** — под профиль: академический / разговорный / деловой
- **Образы и примеры** — из профессионального контекста студента (не абстрактные)
- **Акценты** — на то, что резонирует с его запросом на курс
- **Объём** — достаточный для понимания, не пересказ всего конспекта

Вывести адаптированный текст в чат. Дать студенту время прочитать.  
Спросить: "Всё понятно? Есть вопросы до проверки?"

### Шаг 2 — Проверка усвоения

Только после подтверждения студента — переходить к вопросам из `## Проверка усвоения` в meta.md.

Порядок: сначала уровни 1-2 (Помнит/Понимает), потом — практические задания уровня 3+.  
Задавать вопросы по одному, не списком.  
Фиксировать реальный уровень Блума по ответам студента.

### Шаг 3 — Закрытие чанка

По итогам сессии: обновить `students/[name]/progress.md` (append), `students/[name]/my-path.md` и при наличии attitude-сдвигов — `mindset-map.md`.

### Шаг 4 — Синхронизация с GitHub (обязательно)

После обновления всех файлов — выполнить git-синхронизацию. Это сохраняет прогресс студента в общем репозитории и делает его доступным для всей команды.

Команды для выполнения через bash:
```bash
cd ~/Desktop/AI-M
git pull --rebase origin main
git add -A
git commit -m "[name]: [chunk-id] — [краткое описание сессии]"
git push origin HEAD:main
```

Пример сообщения коммита: `vadim: s3-w1-lecture — DoD минимальный, уровни Блума 1-3`

**Правила:**
- Всегда делать `git pull --rebase` перед пушем — чтобы подтянуть изменения других студентов
- Если `git push` завершился ошибкой — сообщить студенту и попросить разрешить конфликт вручную
- Не пушить файлы из папки `inbox/` — она в `.gitignore`

## Что НЕ делать

- Не редактировать старые записи в `progress.md`
- Не удалять файлы из `raw/` — это архив первичных источников
- Не держать большой системный промт — весь контекст в файлах
- Не создавать чанки без `meta.md` с Блумом
