## Проверка усвоения

### Знание (Knowledge)
- Что является основным ограничением индивидуальной AI-операционной системы, которое приводит к необходимости перехода к командной?
- Назовите два метода сбора командного контекста, упомянутых в юните.

### Понимание (Comprehension)
- Объясните, почему руководство должно принимать на себя "удар" при внедрении новых AI-инструментов в команду.
- Как формализация знаний через правила и скиллы способствует доступности AI-инструментов для всей команды?

### Применение (Application)
- Выберите один из методов сбора контекста и опишите, как вы примените его для опроса сотрудника о его функциях и использовании AI в вашей команде.
- Предложите три конкретных шага для преодоления барьеров внедрения нового AI-скилла для команды, исходя из принципов, описанных в юните.

### Анализ (Analysis)
- Проанализируйте, как выбор метода сбора контекста (например, интервью vs. форма) может повлиять на полноту и качество полученной информации для разных типов проектов.
- Оцените, какие виды "красивых приблуд" (упрощений/улучшений) могут быть наиболее эффективными для повышения адаптации команды к AI-инструментам.

### Синтез (Synthesis)
- Разработайте краткий план для создания первичной базы знаний AI-операционной системы для команды из 5 человек, опираясь на методы сбора контекста.
- Сформулируйте три базовых правила для использования гипотетического AI-инструмента в командном рабочем процессе.

### Оценка (Evaluation)
- Оцените, насколько важен системный подход к AI в команде по сравнению с индивидуальным использованием AI для достижения стратегических целей компании.
- Оцените эффективность предложенного в юните подхода к формализации знаний на примере гипотетической "Партнерской CRM".

## Definition of Done

### Минимальный DoD (Bloom 1-3)
- Список основных принципов перехода к командной AI-операционной системе и методов сбора контекста.
- Проект плана интервью для сбора контекста по одной функциональной области в команде.

### Полный DoD (Bloom 4-6)
- Предложение по внедрению AI-инструмента в небольшую команду, включающее методы сбора контекста, принципы формализации правил и стратегию преодоления барьеров.
- Анализ кейса или сценария внедрения нового AI-инструмента в команду, с оценкой рисков и предложениями по их минимизации.
