## Проверка усвоения

### Знание (Knowledge)
- Назовите три основных типа данных, которые полезны для AI-агентов, и приведите примеры для каждого.
- Что является принципиально нереализуемой задачей при попытке интеграции всех корпоративных знаний для AI-агентов в крупной организации?

### Понимание (Comprehension)
- Объясните, что означает принцип "Источника истины" (Source of Truth) для AI-агента, работающего с данными во внешних системах.
- Опишите, как AI-агент взаимодействует с данными, генерируемыми в рамках его собственного процесса.

### Применение (Application)
- Предложите последовательность шагов для внедрения AI-агента в отделе продаж крупной компании, который должен работать с CRM-системой.
- Смоделируйте, как AI-агент может использовать существующие API внутренних систем для выполнения задачи, не нарушая модель данных и политику доступа.

### Анализ (Analysis)
- Сравните вызовы и стратегии работы AI-агентов с "данными во внешних системах" и "данными во внутренних системах". В чем их ключевые различия?
- Проанализируйте, почему "постепенный роллаут" является предпочтительной стратегией внедрения AI-агентов в крупных организациях, основываясь на примерах из лекции.

### Синтез (Synthesis)
- Разработайте упрощенный сценарий, как AI-агент может эмулировать действия человека для работы с устаревшей системой типа SAP, чтобы автоматизировать рутинный отчет.
- Предложите набор инструкций (клодом ди / HCD) для AI-агента, работающего с внутренней базой данных сотрудников, чтобы обеспечить безопасность и ограничить его действия.

### Оценка (Evaluation)
- Оцените риски и преимущества внедрения AI-агентов в сферах, где существуют строгие регуляторные требования (например, финансы или медицина).
- Обоснуйте, почему человек должен проверять и одобрять действия агента в критически важных бизнес-процессах, даже после успешного тестирования.

## Definition of Done

### Минимальный DoD (Bloom 1-3)
- Студент может перечислить три типа данных для AI-агентов и объяснить принцип "Источника истины" своими словами.
- Студент может описать шаги, которые AI-агент должен выполнить для работы с данными во внешней системе, следуя принципу "Источника истины".

### Полный DoD (Bloom 4-6)
- Студент может проанализировать комплексную задачу интеграции AI-агента с корпоративными системами, предложить стратегию управления данными и разработать план поэтапного внедрения, учитывающий риски и регуляторные требования.
- Студент может критически оценить подходы к масштабированию AI в крупных организациях, предлагая адаптированные решения для конкретных сценариев, включая примеры из лекции.
