## Фокус воркшопа

Визуальный слой: как создавать HTML-дашборды через вайб-кодинг без навыков фронтенд-разработки. Инструменты: Codex, Cursor.

**Ключевой тезис:** дашборд — лучший формат для передачи информации клиенту или команде, чем презентация. HTML-файл открывается в браузере, его можно расшарить ссылкой, выложить на Vercel за минуту.

---

## Техника "5 вопросов перед кодом"

Прежде чем просить агента написать дашборд — задай ему промт:

```
Перед тем как писать код, задай мне 5 вопросов, ответы на которые
сильнее всего повлияют на то, что войдёт в дашборд.
Правила: один вопрос за одно сообщение, по очереди.
Не угадывай — спрашивай.
После моих ответов покажи вопрос с тремя вариантами ответов на каждый.
```

Это техника борьбы с "белым листом": агент сам помогает тебе сформулировать, что именно ты хочешь, прежде чем начать делать.

## HTML как рабочий артефакт

**HTML-файл** — базовый артефакт визуализации в AI-работе:
- Открывается в браузере без установки ничего
- Легко передаётся
- Просто выкладывается онлайн (Vercel, Netlify — бесплатно, за минуты)
- Агенты генерируют его хорошо
- Tailwind CSS — простой и понятный агентам способ стилизации

**Markdown** — более простой вариант для структурированного текста. HTML — для визуализации с интерактивностью.

## Организация файлов и GitHub

**Рабочая папка:** создай папку для каждого проекта. Используй префиксы в названиях папок (`corp_`, `mktg_`, `ai_`) — потом легко искать и говорить агенту "посмотри в папках с префиксом corp_".

**GitHub** — синхронизация файлов как Dropbox для кода. Бесплатен. Текст/код = текст, поэтому хранить там все рабочие файлы — правильно. Это важно для будущего перехода на мультиюзерную работу.

## Жанр воркшопа: live-кодинг

Основной метод — делать вместе в реальном времени:
1. Спикер показывает свой экран
2. Участники повторяют в своей среде (Codex / Cursor / Claude Code)
3. Вопросы — в чат, спикер отвечает каждые 15 минут

Если что-то не получается — смотри запись в LMS или пре-реквизиты (видео ~30 минут).
