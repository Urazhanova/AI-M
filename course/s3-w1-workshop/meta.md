## Проверка усвоения

### Знание (Knowledge)
- Какие два основных вызова упомянуты в начале воркшопа при внедрении AI-native подходов, несмотря на обилие инструментов?
- Перечислите как минимум два примера инструментов или методов, которые участники воркшопа уже опробовали в своих AI-native экспериментах.

### Понимание (Comprehension)
- Объясните своими словами, почему "думать все равно надо", даже когда LLM ведет пользователя по всем шагам рабочего процесса.
- Приведите пример, как можно снять барьер внедрения нового инструмента для сбора контекста у сотрудников, используя подход, предложенный в воркшопе.

### Применение (Application)
- Разработайте краткий план из 3-4 шагов, как вы бы использовали LLM для ведения пользователя по процессу подачи заявки, основываясь на описании этого процесса на лендинге.
- Предложите структуру (набор ключевых вопросов или разделов) для часового интервью с сотрудником с целью сбора контекста о его проекте.

### Анализ (Analysis)
- Проанализируйте, почему при создании инструмента для интервьюирования сотрудников "основная сложность заключается в согласовании сотрудников относительно общих целей и ценностей", а не в технической реализации.
- Сравните плюсы и минусы использования Google Form/Tally Form для сбора контекста по сравнению с личным интервью.

### Синтез (Synthesis)
- Разработайте мини-стратегию для инициатора проекта по внедрению нового инструмента сбора контекста, чтобы минимизировать сопротивление и обеспечить вовлечение сотрудников.
- Предложите, как можно комбинировать личные интервью и автоматизированные формы для наиболее эффективного сбора контекста о проектах.

### Оценка (Evaluation)
- Оцените, насколько критично для успеха внедрения нового AI-native инструмента, чтобы инициатор проекта взял на себя ответственность за обучение и внедрение. Обоснуйте свой ответ.
- Дайте оценку утверждению: "Идеальный мир, где все всё сразу понимают, не существует, поэтому нужны красивые и функциональные "приблуды"". Согласны ли вы? Почему?

## Definition of Done

### Минимальный DoD (Bloom 1-3)
- Студент может назвать основные вызовы, связанные с концентрацией и фокусом при внедрении AI-native инструментов.
- Студент может описать, как LLM-бот может быть использован для ведения пользователя по рабочему процессу.

### Полный DoD (Bloom 4-6)
- Студент способен разработать фреймворк для сбора контекста о проектах от сотрудников, включая методы вовлечения и снятия барьеров.
- Студент может критически оценить роль личной ответственности и дизайна инструмента в успешном внедрении AI-native решений.
