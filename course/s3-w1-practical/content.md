## Главная идея сессии

Большинство людей при попытке работать с AI думают об инструменте (какого агента выбрать, какую модель), а не о задаче. Это ошибка. Пока не сформулирована задача — не важно, какой агент.

**Тезис:** агент — идеальный исполнитель, но максимально "тупой". Он делает ровно то, что ему сказано, в тех ограничениях, которые заданы. Он не догадается сам о контексте, не поймёт корпоративную субординацию, не прочитает между строк.

---

## Пять компонентов формализации задачи

Прежде чем запускать агента, нужно ответить на 5 вопросов:

1. **Что нужно сделать?** — конкретный результат, не абстракция
2. **Что нужно для этого?** — данные, правила, ограничения, контекст
3. **Как это обрабатывать?** — сценарий, алгоритм
4. **Что должно быть на выходе?** — конкретный формат результата
5. **Как оценить, что результат хороший?** — критерии оценки, референтный стандарт

Без пункта 5 невозможно улучшаться. Это главная незакрытая точка у большинства.

## Контекст vs. Контент

**Контент** — всё, что у вас есть (весь массив информации).  
**Контекст** — минимально необходимая и достаточная часть контента для решения конкретной задачи.

LLM работает с контекстом так же, как человек: у нас 7 объектов в оперативной памяти одновременно. Чем больше "контента" вы грузите в модель, тем больше шума и хуже результат.

> Хороший контекст — минимальный по объёму, максимальный по релевантности для задачи.

**Откуда берётся контекст:**
- Системный промт (кто агент, что его ограничивает)
- Файлы (локальные, подключённые через MCP)
- История чата
- Инструменты (что агент может вызвать)

**Правило:** системный промт лучше держать пустым или минимальным. Все данные — в отдельных файлах, которые подгружаются под конкретную задачу.

## Правила для агентов

Ошибка: писать абстрактные правила ("делай хорошо", "не ошибайся", "будь профессиональным").  
Правильно: конкретные ограничения для конкретной задачи.

**Пример правильного правила:**
> "Дай резюме. Затем дай выводы в формате [такой-то]. Затем опиши риски с приоритетами. Затем — следующие шаги. Если данных не хватает — не выдумывай, укажи пробел."

5-8 конкретных ограничений на задачу — это не корпоративная политика, это рабочий инструмент.

---

## Карта источников контекста

Для каждой задачи полезно составить карту:
- Откуда берётся контекст (транскрипт, заметка, шаблон отчёта, календарь)
- Уровень безопасности каждого источника (есть ли там персональные данные, пароли?)
- Используется ли он сейчас или только планируется

Это не бюрократия — это способ не утонуть и не "сливать" данные случайно.

---

## Работа с большими документами

Задача "прочитай 200 страниц и выдели главное" — плохо работает. Правильный подход:  
1. Каталогизировать документ (оглавление, структура)
2. Определить, в какую секцию нужно "провалиться"
3. Работать с конкретной релевантной частью

Так работает человек с большим документом — агент не исключение.

## Артефакты первой недели (домашняя работа)

По итогам практической сессии у каждого ученика должны появиться:
1. **Описание конкретной рутинной задачи** (формализованной по 5 вопросам)
2. **Список источников контекста** (откуда берётся информация для этой задачи)
3. **Набор правил** для обработки этой задачи
4. **Первый улучшенный результат** работы с агентом по этим правилам

## Эффект Гудхарта в AI-работе

Когда метрикой становится количество использованных токенов или сгенерированного кода — оптимизируется метрика, а не качество работы. Важно: KPI "AI-чемпионов" в компании должен быть завязан на бизнес-результат, а не на использование инструмента.
